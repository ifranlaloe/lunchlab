# AGENTS.md

This repository contains slide decks for LunchLab sessions.

## Working norms
- Keep deck content practical and facilitation-oriented.
- Prefer conceptual consistency across a deck over adding many disconnected examples.
- When updating session guidance, reflect changes in the session README and keep language concise.

## LunchLab #2 concept guardrails
For `session-2-api-styles/`, use this teaching concept:

1. Teach C4 with an infrastructure-cartography metaphor that stays consistent across all levels.
2. Use the Netherlands map zoom journey to explain C4 abstraction changes:
   - Level 1: national/regional boundary interactions.
   - Level 2: city/container routes inside the system boundary.
   - Level 3: building/component organization.
   - Level 4: blueprint/code details.
3. At Level 1, emphasize multiple external route modalities to explain boundary integration constraints:
   - sea, river, air, truck, and rail.
4. Reuse the metaphor at subsequent levels so participants map physical infrastructure choices to IT/API infrastructure choices.
5. Keep workshop flow as:
   - zoom-level explanation first,
   - room puzzle on food-delivery case second,
   - prepared solution slides as facilitator backup only.
6. Preserve C4 scope discipline for the main session:
   - prioritize Level 1 and Level 2 decisions,
   - treat Level 3 and Level 4 as useful but mostly out-of-scope for this lunch-and-learn.
